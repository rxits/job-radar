# job-radar v3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add LinkedIn + X sources, internship/region/pay classification, ranking, contact enrichment, and best-match auto-tailoring to the existing `rxits/job-radar` repo — in place.

**Architecture:** Extend the existing pluggable `JobSource` system and SQLite layer. New deterministic classifiers run at scrape/normalize time; new sources register in `sources/index.ts`; contact enrichment + auto-tailoring are new modules that reuse the existing Gemini client and kit generator. Migration `user_version` 3 → 4.

**Tech Stack:** Next.js 15, TypeScript, better-sqlite3, @google/genai (Gemini), vitest. No live network in tests — fixtures in `test/fixtures/`.

## Global Constraints

- Runtime AI is Gemini only (`GEMINI_API_KEY`); `FLASH = gemini-2.5-flash`, `PRO = gemini-2.5-pro`.
- All SQL lives in `src/lib/db.ts`. Sources implement `JobSource` and register in `src/lib/sources/index.ts`.
- No live network in tests. `pnpm test` green + `pnpm exec tsc --noEmit` clean before every commit.
- `data/` is gitignored — never commit the DB or profile.
- No paid APIs, no logged-in LinkedIn, no auto-send of applications.
- Truthful-re-emphasis rules in kit generation preserved — never fabricate experience.
